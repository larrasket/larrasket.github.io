Thursday, 03 November 2022

I do 'programming' much more than software engineering. In other words, I write programs for myself to use more frequently than I develop software that will be used for some business, and you probably do the same thing as me. My question is, what is the most appropriate language to do so? The answer is the most language that was made for programmer rather than software engineers. There is no such a language, but there is a language that is not made for either; lisp! I regret writing Go programs for my use, I regret the time I wasted 'design' and 'test', when I could just open my editor and start to write with Lisp. I'm saying that because I'm now in the middle of a personal project I started in Go, and now I've to stop programming every 5 minutes, thinking about test cases, project architecture and the best design. I want a language that I can misuse!
