I booted this morning to Windows 10, which I had to install for a scientific purpose (testing malware), and after I finished work and restart (to boot to my main OS) I found out that there was a 'Windows update' that fucked up my grub configurations. GRUB couldn't be found anymore in the boot entries, however I was able to locate it at /EFI/ just right to Windows. I don't know much about grub but I think this was only resolvable by a grub-install process. Which is possible using a live USB running a grub instance. However, I didn't have any of my memory drivers since I wasn't home, I was completely desperate that I will have to wait until I find a USB and burn it. Felt terrible. Occasionally I remembered the bcdedit, Windows' sucky editor boot configuration data, and I found that you can do something like: 

bcdedit /set "{X}" path "\EFI\path\to\app.efi"

is possible to be executed, so I said: why not change the windows boot manager path and link it to grub instead? So, we make windows entry point to grub instead! Sounded like a crazy idea, but why not?

I had to know the path for my grub, (I could anticipate it, but when dealing with boot configurations, mistakes are expenses). Windows doesn't let you to open /EFI, however there is a trick: 

mountvol U: /S
U:

Now, browse the EFI partition and find the grub EFI location. Replace it in the antecedent command. And done!

It wasn't a big deal but I felt great that I could do it without reinstalling grub.
